放假第一周，王爽的《汇编语言》（第三版）看完了前三章，本文是总结的第一部分，包括了书上的检测点答案

第一章 基础知识

检测点 1.1

  • 1个CPU的寻址能力为8KB，那么它的地址总线的宽度为____。

  • 1KB的存储器有__ 个存储单元。存储单元的编号从 _ 到。

  • 1KB的存储器可以存储_个bit，____个Byte。

  • 1GB、1MB、1KB分别是____Byte

  • 8080、8088、80286、80386的地址总线宽度分别为16根、20根、24根、32根，则他们的寻址能力分别为_（KB）、_（MB）、_（MB）、____（GB）。

  • 8080、8088、8086、80286、80386的数据总线宽度分别为8根、8根、16根、16根、32根。则它们一次可以传送的数据为：_（B）、_（B）、_（B）、_（B）、____（B）。

  • 从内存中读取1024字节的数据，8086至少要读取_次，80386至少要读取____次。

  • 在存储器中，数据和程序以____形式存放。

    解析：（1）CPU在内存中寻址的最小单位是Byte（字节），8KB = 2^13B，因此地址总线的宽度为13.

（2）存储器被划分成若干个存储单元——可以存储一个Byte，每个存储单元从0开始顺序编号。1KB = 2^10 B。因此1KB存储器有1024个存储单元，编号从0到127。

（3）1KB = 1024 Byte = 1024*8 bit = 8192 bit

（4）1GB = 1024 MB = 1024^2 KB = 1024^3 B

（5）一个CPU有N根地址线，则可以说这个CPU的地址总线宽度为N。这样的CPU最多可寻址2的N次方个内存单元。因此8080地址长度为16根即寻址能力为2^16 B = 2^6 KB;8088地址长度为320根即寻址能力为2^20 B = 1 MB;80286地址长度为24根即寻址能力为2^24 B = 2^4 MB;80386地址长度为16根即寻址能力为2^32 B = 2^2 GB;

（6）8根数据总线一次可传送一个8位二进制数据（即一个字节）。因此8080数据总线宽度为8根即一次可传送的数据为 1B；8088数据总线宽度为8根即一次可传送的数据为 1B；8086数据总线宽度为16根即一次可传送的数据为 2B；80286数据总线宽度为16根即一次可传送的数据为 2B；80386数据总线宽度为32根即一次可传送的数据为 4B。

（7）由于8086数据总线宽度为16根，一次传送2B，因此8086至少要读1024/2=512次，80386至少要读1024/4=256次。

（8）在存储器中，数据和程序以二进制形式存放，即0 1。

(2) 只能使用目前学过的汇编指令，最多使用4条指令，编程计算2的4次方。

检测点 2.2

（1）给定段地址为0001H，仅通过变化偏移地址寻址，CPU的寻址范围为_到_____。

解：物理地址＝段地址(SA)*16+偏移地址（EA） 

EA的变化范围为0000h~ffffh   

物理地址范围为(SA*16+0000h)~(SA*16+ffffh)   

现在SA=0001h,那么寻址范围为   

(0001h*16+0h)~(0001h*16+ffffh)   

=00010h~1000fh

（2）有一数据存放在内存20000H的单元中，现给定段地址为SA，若想用偏移地址寻到此单元。则SA应满足的条件是：最小为_，最大为_____。提示，反过来思考一下，当段地址给定为多少，CPU无论怎么变化偏移地址都无法寻到20000H单元？

解：物理地址 = SA*16+EA   

20000h = SA*16+EA   

SA = (20000h-EA)/16 = 2000h-EA/16   

EA取最小值时,SA = 2000h-0h/16 = 2000h,SA为最大值为 2000h

（很多人最大值很容易算出来，但是最小值卡住了，很纳闷为什么是1001H）

SA * 16 + FFFFH = 20000H    段最小值那么就要用最大偏移地址来配合它。
SA * 16 = 20000H - FFFFH
SA * 16 = 10001H    我们求得了这个段地址，但是还要除以10H，或者转换为10进制，再除以10进制的16

(那为什么不是1000H呢？下面是答案……）

段地址要为16(10H)的倍数,

20000H - FFFF = 10001H 10001H十进制是65537，除以16是4096.0625   有小数说明不是倍数，不能整除
20000H - FFFE = 10002H 10002H十进制是65538，除以16是4096.125    有小数说明不是倍数，不能整除
20000H - FFFD = 10003H 10001H十进制是65539，除以16是4096.1875   有小数说明不是倍数，不能整除
20000H - FFFC = 10004H 10001H十进制是65540，除以16是4096.25     有小数说明不是倍数，不能整除
20000H - FFFB = 10005H 10001H十进制是65541，除以16是4096.3125   有小数说明不是倍数，不能整除
20000H - FFFA = 10006H 10001H十进制是65542，除以16是4096.375    有小数说明不是倍数，不能整除
（这些的值都不是16的倍数）

我们不断减小值，因为FFFF是最大，不能再大了，所以只能减小，要直到除以16是整数为止

20000H - FFF0 = 10010H   10010H十进制是65552，除以16是4097，可以整除；这时10010H满足16(10H)的倍数了

所以段最小值 SA = 10010H/10H = 1001H

（3）提示，反过来思考一下，当段地址给定为多少，CPU无论怎么变化偏移地址都无法寻到20000H单元？

根据上面一题的答案，小于1001H，或者大于2000H都无法寻到20000H单元

检测点 2.3

下面的3条指令执行后，CPU几次修改IP？都是在什么时候？最后IP中的值是多少？

mov ax, bx
sub ax, ax
jmp ax

解：一共修改四次
第一次：执行完第一条指令 此时ax=bx，IP修改
第二次：执行完第二条指令。此时AX清零，IP修改
第三次：执行完第三条指令，IP修改
第四次：jmp无条件跳转 此时IP=AX=0
