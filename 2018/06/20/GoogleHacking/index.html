Google搜索通配符

<OR必须大写>

Google通配符*代表一个单词

Google是从左到右读取你的查询

intext:password | passcode intext:username | userid |user filetype:csv这个例子使用布尔运算OR这个高级运算符构造了一个查询，使之读起来像”定位所有在文档的文本中有password或者passcode的页面，从这些页面中，只显示文本档的文本中有单词username或者userid或者user的页面，从这些页面中，只显示是CSV文件的文档”

高级运算符

intitle

  1. 在页面的标题里查找字符串
  2. 与其他运算符混合使用效果佳
  3. 最好在网页、网上论坛、图片和新闻搜索中使用

allintitle

  1. 在一个页面的标题里查找所有搜素项
  2. 其他运算符或搜索项混合使用效果差
  3. 最好在网页、网上论坛、图片和新闻搜索中使用

inurl

  1. 在一个页面的URL里查找字符串
  2. 与其他运算符混合使用效果佳
  3. 最好在网页和图片搜索中使用

allinurl

  1. 在一个页面的URL里查找所有搜索项
  2. 其他运算符或搜索项混合使用效果差
  3. 最好在网页、网上论坛和图片搜索中使用

filetype

  1. 基于文件扩展名搜索特殊类型文件
  2. 与ext同义
  3. 需要一个额外搜索项
  4. 其他运算符或搜索项混合使用效果差
  5. 最好在网页和网上论坛搜索中使用

allintext

  1. 在一个页面的文本里查找所有搜索项
  2. 纯粹一个小恶魔-就别用它
  3. 忘掉所有关于allintext的传说

site

  1. 把一个搜索限定在一个特别的网站或域中
  2. 与其他运算符混合使用效果佳
  3. 能够单独使用
  4. 最好在网页、网上论坛和图片搜索中使用

link

  1. 搜索一个网站的链接或者URL
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

inanchor

  1. 在链接的描述性文字中查找文本
  2. 其他运算符或搜索项混合使用效果佳
  3. 最好在网页、图片和新闻搜索中使用

daterange

  1. 寻找一个特定日期范围内索引过的页面
  2. 需要一个搜索项
  3. 与其他运算符或搜索项混合使用效果佳
  4. 最好在网页搜索中使用
  5. 可能会让位于as_qdr而被淘汰

numrange

  1. 在特定范围内查找一个数字
  2. 与其他运算符或搜索项混合使用效果佳
  3. 最好在网页搜索中使用
  4. 与ext同义

cache

  1. 显示Google缓存的页面拷贝
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

info

  1. 显示关于一个页面的总结信息
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

related

  1. 显示与所给网站或URL相关的站点
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

stocks

  1. 为一个股票代码显示Yahoo财经的股票列表
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

define

  1. 显示单词或短语的各种不同含义
  2. 无法与其他运算符或搜索项混合使用
  3. 最好在网页搜索中使用

Google Hacking基础

缓存的匿名特性

  • 点击缓存链接不仅仅从Google数据库加载页面，它也会连接真实的服务器来访问图表和其他非HTML内容
  • 在一个缓存URL的末尾添加&strip=1则仅仅显示一个缓存页面的HTML。

查找目录列表

  • 目录列表包含了大量的无价信息
  • 瞄准包含目录列表页面的最好方法就是通过类似intitle:index.of “partent directory”或者intitle:index.of name size这样的查询

在列表中查找特定的目录

  • 通过给一个index.of搜索添加一个目录名，你能很容易地在一个目录列表里查找特定目录。例如，intitle:index.of inurl:backup能用来查找在URL里有单词backup的目录列表。如果单词backup在URL里，这是目录名的机会就会很大。

在目录列表中查找特定的文件

  • 通过简单的在index.of查询里添加文件名，你就能在一个文件列表里找到特定的文件，例如intitle:index.of ws_ftp.log

目录列表中的服务器版本信息

  • 一些服务器，特别是Apache及其衍生系列会在一个目录列表的底部添加一个服务器的标签。这些服务器标签能通过把一个index.of搜索扩展成intitle:index.of serve.at来找到，该搜索关注了短语 serve.at。

  • 通过使用更多来扩展这个查询，这些信息由一个正确格式化的服务器标签衍生而来，你能查找到一个web服务器特定的版本信息。例如，查询intitle:index.of serve.at”Apache Tomcat/“将会查找到正在运行的各种版本Apache Tomcat服务器

目录遍历

  • 一旦你已经在目标web服务器上查找到一个特定的目录，你就能使用这个技术找到其他目录或子目录。

  • 完成这个任务的一个简单方法就是通过目录列表。简单的点击parent directory链接，它将退回至当前目录的上一级目录。如果这个目录含有另一个目录列表，你能从简单的从该页面点击链接来探索其他目录。如果父目录没有显示一个目录列表，你可能就得凭借更加复杂的方法，通过猜测目录名并把它们加到父目录URL的尾部。此外，考虑在一个Google搜索中使用site和inurl关键词。

增量替换

  • 增量替换就是“找到一个数字并用更大或更小的数字替换它”的一个时髦的说法。
  • 这种技术可以用来探索一个在目录或文件名中使用了数字的网站。简单地用更大或更小的数字替换它，注意保证剩下的文件或目录名要完全一致（特别注意哪些0！）。此外，考虑在一个创造性的Google搜索中使用带有关键字inurl或者filetype的site搜索。

扩展遍历

  • 这个技术能帮你找到有相同文件名不同扩展名的文件（例如，备份文件）
  • 执行扩展遍历的最简单地方法就是用一个扩展名替换URL里另一个，例如，用bak替换HTML。
  • 目录列表，特别是缓存目录列表，是用来确定备份文件是否存在，以及在网站其他地方会使用何种文件扩展名的简单方法。

文档细分和数据挖掘

配置文件

  • 配置文件会泄露敏感信息给攻击者
  • 尽管名字不同，配置文件通常都能通过类似ini，conf，config或CFG这样的文件扩展名找到

日志文件

  • 配置文件也会泄露敏感信息，而且这些信息通常比配置文件中能找到的信息更新更快
  • 命名规则是不尽相同的，但日志文件通常都能通过类似log这样的文件扩展名找到。

office文档

  • 大多数情况下office文档就是用来公开发布的。那些并非想对公众开放的文档可能就包括敏感信息。
  • 普通的office文件扩展名包括PDF，doc，txt或者XLS
  • 尽管文档内容不同，但类似private，password，backup或admin这样的字符串就能表示其可能为一个敏感文档。

数据库挖掘

  • 登录入口，特别是由软件开发商提供的默认入口，是很容易搜到的，它对攻击者查找特别版本或类型的软件有很强的吸引力。
  • 单词login，welcome和copyright statements都是用来找到登录入口的极好的方法。
  • 服务器和客户端软件上就会存在支持文件。这些文件会泄露配置信息或一个应用的使用方法。
  • 错误信息的不同内容能用来描绘一个目标。
  • 数据库转储无可辩驳是最易泄露整个数据库的全部或部分内容。这些转储能通过在标题中查找类似“#dumping data for table”这样的字符串找到。
